Wikipedia change-tracking

Track watched pages per language edition (editions are per-language, not per-country). We store one baseline snapshot per page, then only diffs/deltas of new edits — so cosmetic changes cost almost nothing — and flag large / suspicious edits (size delta, revert/blank tags, anonymous, bursts, optional ORES). Candidates are surfaced for you to judge; nothing is labelled "disinformation".

Keyword filtering

"Dumb" keywords (function words like the, you, not, one) are removed automatically by a built-in multilingual stoplist. Fine-tune below: exclude extra terms (one per line or comma-separated), set the minimum length, and toggle options. Excluding hides a keyword everywhere (reversibly — stored mentions are kept). You can also click the ✕ next to any keyword in Insights.

Wikipedia offline baselines

Optional, heavy, and separate from live change-tracking. Pick a language edition below and download the official Wikimedia current-text dump (resumable, with pause/resume). Editions are grouped by continent (largest-first within each); the real download size is read from the server when you click Estimate size — nothing is guessed. enwiki ≈ tens of GB; most editions are far smaller. Baselines are only for offline reading/search — change-tracking (Wikipedia tab) does not need them.

The list covers 100+ editions, grouped by continent (largest first within each) — type in the box to filter by name, autonym or code. Sizes shown are coarse heaviness hints, while Estimate size reports the exact current dump size from the server. You can also type any edition code that isn't listed.

Backup & restore

Checking backend…
A consistent snapshot of the entire corpus, taken live via SQLite's online-backup API.
Restore replaces your current corpus with the uploaded file. The file is validated first (must be a genuine Open Omniscience SQLite database), and your current corpus is snapshotted to a pre-restore-*.db beside the database so the operation is reversible.
