Source integrity & anti-amplification

The app measures structure — coordination, novelty, output capacity, transparency — and you decide. There is deliberately no trust score: a single number over incommensurable dimensions would bake in bias and silence small, foreign, new or dissident sources. Surface, never suppress.

Keyword filtering

"Dumb" keywords (function words like the, you, not, one) are removed automatically by a built-in multilingual stoplist. Fine-tune below: exclude extra terms (one per line or comma-separated), set the minimum length, and toggle options. Excluding hides a keyword everywhere (reversibly — stored mentions are kept). You can also click the ✕ next to any keyword in Insights.

Local models (Ollama)

The local language models power Summarize, Translate and Synthesize. This shows what you actually have installed (live from Ollama) and a small suggested list annotated for your hardware. Pick the active model, pull new ones, or remove them — all from here. Newer and better models appear constantly; browse ollama.com/library and pull any tag below.

Pulling a model is a network action: it downloads through the Ollama process over the clear internet — not through this app's Tor proxy — so it is refused while airplane mode is on, and asks your consent the first time you go online. The download happens on your machine; the model then runs fully offline.

One file carries everything: articles, keywords, Wikipedia and law snapshots, market series, events, settings, annotations and the custody log — with a signed manifest, so tampering is detectable. Encrypted with a passphrase by default; there is no recovery for a lost passphrase.

Safety & at-risk use

For journalists working under pressure. These tools are honest about their limits — read each note. Everything stays on this machine.

At-rest encryption

Checking…

At-rest encryption protects a seized or copied file. It cannot protect a compromised running session.

Encrypt this corpus…
Choose something unique and remember it. There is no recovery and no decryption alternative. A plaintext snapshot is kept as your escape hatch — delete it once you have unlocked successfully.

Encrypted backup

An encrypted snapshot of the whole corpus (AES-256-GCM + scrypt). A lost or wrong passphrase means the file cannot be opened — there is no recovery. Keep the passphrase safe.

Restore is additive-only — the destructive replace-restore was removed. Restore through the merge restore above (the signed backup artifact), which complements your corpus and never overwrites it.

Database maintenance

Compacting rebuilds the database file: it reclaims the space deletions leave behind and defragments the indexes. It takes time proportional to the file size and pauses collection writes while it runs — nothing is interpreted or removed.

Corpus integrity (fixity)

Re-hash every stored article and compare to the hash recorded when it was captured. A mismatch means the stored content no longer matches its capture-time hash — evidence of tampering or bit-rot. Nothing is changed; the check is local and read-only.

Network fetch mode

Transparent mode names this tool in the User-Agent (polite, but identifiable). Protected mode sends a generic User-Agent and routes every fetch through a proxy you run.

Protected mode cannot guarantee anonymity — you must run and trust the proxy yourself. SOCKS proxies need the optional [safety] extra.

External topic discovery

The one feature that contacts an external service: Sources → Discover by topic sends your topic query to DuckDuckGo to find candidate outlets. Your query leaves this machine. Off by default; everything else works without it. (Discovering RSS feeds for sources you added yourself stays local-path and is not affected.)

Panic wipe

Irreversibly deletes the corpus, keys and caches on this machine, then the app must be restarted. This cannot be undone. Overwrite-in-place does not guarantee unrecoverability on SSD/flash — for that, use full-disk encryption (LUKS/Qubes/Tails) and destroy the key.

Uninstall the app

Removes the app's virtualenv and its desktop launchers, then stops the server. Your data is kept (to destroy the corpus/keys, use Panic wipe above first). The app folder itself is left in place — delete it manually if you want it gone. You can also run ./install.sh --uninstall in a terminal, or use the “Uninstall Open Omniscience” desktop icon.
