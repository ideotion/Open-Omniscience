Behaviour & prompts

There are four system prompts — Summary (used for one or many articles), Translation (one or many), Synthesis (one combined output across several), and Keyword extraction (the built-in keyword/entity indexer). They instruct the local model, and the exact prompt used is recorded with every result.

How long Ollama keeps the model in memory after a request, so the next one does not reload it. Examples: 30m, 1h, -1 (never unload), 0 (unload now).
Leave a prompt empty to use the built-in default (shown as the placeholder). In the translation prompt, the placeholder {target} is replaced by the target language; in the keyword-extraction prompt, {max_terms} is the per-article limit.

Detect unknown languages (AI)

Some articles arrive with no language tag, and the fast offline detector cannot always classify them (short text, low confidence, or a language it does not cover). This OPT-IN pass asks the local model to identify the language of those still-unknown articles. It is AI-derived and unreliable — a THIRD, clearly-labelled provenance class shown inline in the article view. It NEVER overwrites the source-asserted or offline-deduced language, and a garbage or unclear answer stores nothing. It runs as a cancellable background job (watch it in the task manager), never during scraping, and only on this machine's local model.

Official figures

Fetch a published figure from a documented official machine endpoint (the World Bank API, or a Eurostat / SDMX-JSON dataset). Each value is stored with its full provenance trail and a first-class vintage — a re-fetch later is a new row, never an overwrite, so revisions are preserved as evidence. Producers are compared side by side, never averaged, and a cell mixing different units / seasonal adjustment / base years is flagged, not reconciled. No score is ever computed. This is the one networked action here: the fetch egresses over your configured transport and is refused while airplane mode is on.

Point to a folder ON THIS MACHINE and every .eml under it is imported as a pausable background job — for a 20 GB+ archive the file picker can't handle. Same anonymise-at-ingest path, zero network. Already-imported messages are skipped, so you can pause and resume safely.

Import documents (PDF)

Import your own PDF files into your corpus — the text is extracted and each document is searched, analysed and correlated like any article, and kept as a separate, filterable source ("Imported PDFs") so it never blurs with the web.

Zero network: each PDF is read and its text extracted locally — nothing is ever fetched. A born-digital PDF uses its exact text layer. A scanned / image-only PDF has no text layer: if OCR is installed (the [ocr] extra + the Tesseract binary) it is read via OCR and clearly marked OCR-derived (lower-trust — may contain recognition errors; the original PDF stays the source of truth), otherwise it is skipped with an honest reason. Re-importing the same document is skipped (deduplicated).

Import a whole folder

Point to a folder ON THIS MACHINE and every PDF under it is imported. Same local, zero-network path; already-imported documents are skipped.

Pull from a mailbox (IMAP/POP3)

Pull newsletters live from a mailbox instead of exporting files one by one. Each message goes through the same anonymise-at-ingest path as the file import — the recipient is never stored, no raw message is retained, and tracking links are de-toxed and never followed.

This is a network action. It is refused under airplane mode, and connects to your mail provider directly over TLS (revealing your IP to that provider, like any email client) — it is not routed through Tor. Credentials are used for this one fetch and not stored.

Remove imported newsletters

Delete every imported-newsletter article (.eml + mailbox) from your corpus — for replacing a faulty set with a cleaned re-import. Restoring a backup is additive, so leaving newsletters out of a backup never removes them from the live corpus; this does.

This cannot be undone except from a backup. It removes the articles and all their analysis (keywords, dates, places); the empty newsletter source stays so a clean re-import re-attaches.

Encrypted backup

An encrypted snapshot of the whole corpus (AES-256-GCM + scrypt). A lost or wrong passphrase means the file cannot be opened — there is no recovery. Keep the passphrase safe.

This single-file download is capped at about 2 GiB. For a larger corpus use the Export / Back up… flow (Settings → Data & backup), which streams to a folder with no size limit.

Restore is additive-only — the destructive replace-restore was removed. Restore through the merge restore above (the signed backup artifact), which complements your corpus and never overwrites it.

Database maintenance

Compacting rebuilds the database file: it reclaims the space deletions leave behind and defragments the indexes. It takes time proportional to the file size and pauses collection writes while it runs — nothing is interpreted or removed.

Corpus integrity (fixity)

Re-hash every stored article and compare to the hash recorded when it was captured. A mismatch means the stored content no longer matches its capture-time hash — evidence of tampering or bit-rot. Nothing is changed; the check is local and read-only.

Network fetch mode

Transparent mode names this tool in the User-Agent (polite, but identifiable). Protected mode sends a generic User-Agent and routes every fetch through a proxy you run.

Protected mode cannot guarantee anonymity — you must run and trust the proxy yourself. SOCKS proxies need the optional [safety] extra.

External topic discovery

The one feature that sends a free-text search query to an external service: Settings → External topic discovery sends your topic query to DuckDuckGo to find candidate outlets. Your query leaves this machine. Off by default; everything else works without it. (Discovering RSS feeds for sources you added yourself stays local-path and is not affected.)

Panic wipe

Irreversibly destroys the corpus's encryption key (crypto-erase) and removes the data, keys and caches on this machine — the encrypted store becomes permanently unrecoverable at any size, then the app must be restarted. This cannot be undone. You'll be offered an optional full free-space overwrite afterward as defence-in-depth (that pass alone does not guarantee erasure on SSD/flash/copy-on-write disks — the real guarantee is the destroyed key).

Uninstall the app

The virtualenv and desktop launchers are always removed; whether the app folder and your data go too depends on the mode. You can also run it in a terminal: ./install.sh --uninstall
