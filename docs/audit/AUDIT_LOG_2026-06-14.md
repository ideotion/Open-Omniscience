## 2. Executive summary

**Overall health verdict: GOOD / release-track.** Open-Omniscience 0.09 is a
disciplined, local-first, ethics-by-construction codebase. The five constitution
invariants are upheld in the live code (one with a "by-default" caveat). The
single, mandatory `EthicalFetcher` is a genuinely strong robots-fail-closed +
SSRF-guarded + kill-switched fetch path; the no-composite-score honesty rule is
mechanically enforced at import time; data-layer transaction safety, CSV
formula-injection defense, SQL parameterization, and the migration chain are all
sound; CI gates tests/mypy-ratchet/bandit/sqlcipher across the matrix.

No **S0 blockers** and no outright constitution **violations** were found. The
findings cluster as: (a) **documentation/honesty lag** (docs describing removed
endpoints, stale counts, future-tense "not functional" leftovers in ETHICS.md),
(b) **frontend hardening/CSP debt** (199 inline handlers, `unsafe-inline` CSP,
a11y modal focus), (c) **a few security defense-in-depth gaps** (robots-redirect
SSRF, DNS TOCTOU), and (d) **CI breadth gaps** (no i18n gate; ruff/win-mac
non-blocking).

### Top 10 risks (plain language)

1. **ETHICS.md still says the software is future/"when it becomes functional"** in
   its GPLv3 and dependencies sections — contradicting the file's own corrected
   banner and the shipped reality; it's the project's own honesty doc. *(OO-D14-001, S1)*
2. **The privacy headline "everything stays on this machine — no cloud, no
   telemetry" is unqualified** while Proxy/Tor fetch modes demonstrably send
   traffic off-box. True for the app's own behavior; over-broad as written.
   *(OO-D3-002, S1 — awaits maintainer ruling, carried from audit-06 U1)*
3. **robots.txt fetch follows redirects without per-hop SSRF re-validation**, while
   the page-fetch path carefully does — a blind-SSRF defense-in-depth gap.
   *(OO-D2-001, S2)*
4. **Docs describe a `POST /api/database/restore` that was removed** (additive-only
   restore ruling); README + ARCHITECTURE both still list it. *(OO-D14-003, S2)*
5. **No i18n/locale-completeness gate in CI** despite the ×12-locale informed-
   consent mandate — locale drift can land unblocked. *(OO-D15-001, S2)*
6. **199 inline `onclick` handlers + a `script-src 'unsafe-inline'` CSP** mean the
   XSS backstop is weak; current escaping is correct, but the surface is fragile.
   *(OO-D12-001 + OO-D2-002, S2)*
7. **Tamper-evident custody trail is opt-in by default** (`custody_on_ingest=False`),
   so "full auditability" is not the out-of-box state. *(OO-D5-001, S2)*
8. **Command-palette & task-manager dialogs lack `aria-modal`/focus return** —
   keyboard/screen-reader users tab into the obscured page. *(OO-D13-001, S2)*
9. **100k-article scale paths are unmeasured** (briefing recompute, corpus windows,
   FTS rebuild) — known unknown carried from audit-07. *(OO-D8-001, S2)*
10. **README source count "~2,100+" understates the real 3,205** and is internally
    inconsistent with QUICKSTART. *(OO-D14-002, S2)*

### D2 — Security

#### OO-D2-001 — robots.txt fetch follows redirects without per-hop SSRF re-validation
- **Dimension:** D2 (also D4) · **Severity:** S2 · **Impact:** Medium · **Likelihood:** Possible ·
  **Effort:** S · **Confidence:** High · **Priority:** P2 · **Status:** Open · **Invariant:** Ethical scraping / Operator privacy
- **Evidence:** `src/ingest/__init__.py::EthicalFetcher._get_robots` (line ~515):
  `robots_kwargs = {"timeout": self.timeout, "allow_redirects": True}` then
  `resp = self.session.get(robots_url, **robots_kwargs)`. The page-fetch path
  `_http_get` (line ~418) instead does manual, bounded redirects and re-runs
  `self._guard_target(p.hostname)` on **every** hop; the robots path does not.
- **Observation vs inference:** *Observed* `allow_redirects=True` with no `_guard_target`
  on the robots redirect chain. *Inferred* that a malicious/compromised catalog host
  whose `/robots.txt` 30x-redirects to `127.0.0.1`/`169.254.169.254` would cause a
  blind GET to that internal address (no body returned to the user, but the request is
  made; timing/fail-closed behavior is observable). The *initial* robots URL is on the
  already-guarded public host; only the redirect targets are unguarded.
- **Reproduction:** trace `_get_robots` → `session.get(..., allow_redirects=True)`; no
  IP re-check on `Location`.
- **Expected vs actual:** Expected: robots fetch SSRF-guarded like the page fetch.
  Actual: robots redirects bypass the guard.
- **Remediation:** fetch robots with `allow_redirects=False` and follow hops through the
  same guarded loop as `_http_get` (or cap to 0 redirects for robots).
- **Autonomy:** Needs-operator-sign-off (security boundary, behavior change). **Deps:** none.
  **Refs:** ETH-01 (discovery SSRF, fixed); TEST-03 (page-path rebinding, fixed).

#### OO-D2-002 — Content-Security-Policy allows `script-src 'unsafe-inline'`
- **Dimension:** D2 (also D12) · **Severity:** S2 · **Impact:** Medium · **Likelihood:** Possible ·
  **Effort:** L · **Confidence:** High · **Priority:** P3 · **Status:** Open · **Invariant:** none
- **Evidence:** `src/api/main.py` `_CSP` (line ~338):
  `"script-src 'self' 'unsafe-inline'; "` with the inline comment
  `# UI is inline-heavy; nonce-based CSP is future work`.
- **Observation vs inference:** *Observed* the `'unsafe-inline'` allowance. *Inferred*
  that the CSP cannot serve as an XSS backstop while inline scripts/handlers are allowed
  — it is coupled to the 199-inline-handler debt (OO-D12-001). Current escaping is
  correct (no live XSS sink found), so this is defense-in-depth, not an active hole.
- **Reproduction:** read `_CSP`; cross-ref the 199 `onclick=` count.
- **Remediation:** retire inline handlers (OO-D12-001) → switch to a nonce/hash-based
  `script-src` and drop `'unsafe-inline'`. Sequenced after the inline-handler sweep.
- **Autonomy:** Discuss-before-touching (large UI change; pairs with D12-001). **Deps:** OO-D12-001.

#### OO-D2-003 — Residual DNS-rebinding TOCTOU between SSRF guard and connect
- **Dimension:** D2 · **Severity:** S3 · **Impact:** Low · **Likelihood:** Rare · **Effort:** M ·
  **Confidence:** High · **Priority:** P4 · **Status:** Open (known) · **Invariant:** Operator privacy
- **Evidence:** `src/ingest/__init__.py::_guard_target` resolves the host with
  `socket.getaddrinfo` and checks IPs, then `requests`/`session.get` re-resolves at

- **Remediation:** pin the validated IP into the connection (custom adapter / connect to

- **Autonomy:** Needs-operator-sign-off. **Deps:** none.

- **Dimension:** D5 · **Severity:** S2 · **Impact:** Medium · **Likelihood:** Likely ·
